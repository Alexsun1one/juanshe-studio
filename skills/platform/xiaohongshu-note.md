# Skill: 小红书笔记(种草/观点/干货)

> **参考**:小红书创作服务平台公开能力与笔记生态规则;本文件为原创归纳,不复制平台文案。

## Purpose
把一个观点、经验或产品信息改造成适合小红书移动端阅读的笔记:第一眼能停住、前三行能让人继续看、正文能收藏、结尾能自然互动。它不是缩短版公众号,而是"可截图、可保存、可转发"的移动笔记。

## Applies To
- 平台:小红书图文笔记 / 长笔记正文
- 内容类型:个人经验、方法清单、避坑、产品体验、观点浓缩、书影音/工具推荐
- 可叠加风格:`style/de-ai-tone`

## Input
- 主题 / brief
- 目标人群、使用场景、账号语气
- 可核实的事实、亲历素材或产品信息

## Output
- 标题:12-22 字,一个具体收益 / 痛点 / 反差,不硬凑悬念。
- 前 3 行:直接给问题、结论或反差;不要铺垫背景。
- 正文:短段 + 少量列表 + 1 个可收藏结构。
- 标签:5-8 个,末尾集中放,含 2-3 个搜索词、2-3 个场景词、1-2 个账号定位词。

## Rules
- 第一屏只回答一个问题:为什么读者现在要停下来读。
- 每段 1-3 行;连续列表不超过 6 条;长解释拆成"场景 → 做法 → 结果/代价"。
- 必须有一个"收藏理由":清单、步骤、模板、对照表、避坑点或可复用句式。
- 允许少量 emoji 做导航,但总量控制在 2-6 个;不要把每行都装饰成表情包。
- 个人经验必须诚实:没有亲历就写"我观察到/我会这样判断",不要编造"我亲测"。
- 观点要有颗粒度:少说"提升效率/改善体验",多写"少点哪一步、少等多久、少踩哪个坑"。
- 结尾 CTA 轻一点:提问、让读者补充、收藏提醒即可;不要硬广式催促。

## Structure Patterns
**经验避坑**
1. 痛点或反差开头。
2. 3-5 个坑,每个坑包含"为什么错 + 怎么改"。
3. 一句可保存总结。

**方法清单**
1. 直接给适用人群。
2. 按步骤列,每步只有一个动作。
3. 最后给使用场景或下一步。

**观点笔记**
1. 先给判断。
2. 用 2-3 个具体观察撑住。
3. 给一个和读者相关的行动建议。

## Layout Rules
- plainText 是主产物;不要输出复杂 Markdown 卡片、表格或嵌套块。
- 小标题用短句,可用 `✨` / `📌` / `✅` 少量引导。
- 标签只放末尾,不要夹在正文中间。
- 不要把公众号式长段、引用框、复杂 callout 搬进小红书。

## De-AI Checks
- 删"在这个快节奏时代/值得注意的是/不仅如此/总的来说"。
- 删空泛价值词,替换成具体动作、数字、场景或限制。
- 打散模板句式:不要每段都用"如果你...,那么..."。
- 避免假亲密:"姐妹们冲"只在账号本来就是这种语气时使用。

## Evaluation
- 标题是否能让目标读者 1 秒内知道收益?
- 前 3 行是否不用背景也能成立?
- 正文是否有一个值得收藏的结构?
- 标签是否覆盖搜索词 + 场景词,而不是随手堆热门词?
- 读完是否像真人经验,不是公众号摘要改短?
