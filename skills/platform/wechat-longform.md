# Skill: 公众号长文(深度/观点/故事)

> **参考**:流程思路参考 [oaker-io/wewrite](https://github.com/oaker-io/wewrite)(MIT,公众号全流程 Skill)。
> 排版由代码渲染器负责:`packages/core/src/platforms/wechat/renderer.ts`(全内联样式,粘贴即精排)。

## Purpose
把一篇结构化内容产出为**适合微信公众号**的长文:长读、观点清晰、段落有呼吸、重点突出、排版精致。

## Applies To
- 平台:微信公众号
- 内容类型:深度观点、系统论述、故事、品牌内容、方法论
- 可叠加风格:`style/kazike-narrative`、`style/de-ai-tone`

## Input
- ContentDocument(标题 + 结构化 blocks)
- 账号画像(语气/受众/禁忌,若有)

## Output
- 正文(交给 WechatRenderer → 全内联样式 HTML)
- 标题(主标题 ≤ 20 字更佳)+ 摘要(digest,≤ 120 字)
- 封面建议(image_slot, purpose=cover)

## Rules
- 首屏(标题 + 摘要 + 前 3 段)必须抓人;开头忌"在当今 XX 时代"式。
- 前 3 段必须完成一个闭环:提出具体问题 / 给出反常判断 / 把读者拉进场景。不要只交代背景。
- 段落短、留白多;普通段 60-180 字,关键句可独立成段;避免连续 3 段同长度。
- 每 600-900 字设置一个二级小标题(H2)或视觉换气点,让读者能扫读后回到正文。
- 一篇长文保留 2-4 个视觉重点:引用块、callout、step、highlight、表格、分割线任选;不要把每段都做成卡片。
- 结尾给一个明确 CTA(关注/留言/转发/收藏),但 CTA 必须顺着文章结论自然落下,不要像广告口播。
- 标题最好 12-24 字;摘要 digest ≤ 120 字,写文章真正的钩子,不要复述标题。

## Writing Craft
- 观点文:每个判断都要接一个"为什么现在重要 / 读者该怎么用"。
- 故事文:用具体场景开头,让人物、动作、物件先出现,再上观点。
- 方法文:步骤不要超过 6 个;每步写"动作 + 判断标准 + 常见误区"。
- 资料型长文:事实、案例、引用要标明来源线索;不确定的地方明确写"需核实"。
- 去 AI 味:删套话总括,把"趋势/变化/赋能/闭环"换成读者能看见的动作、成本和收益。

## Style Tokens
靛蓝主色(与产品一致)、H2 左侧色条、引用块浅底左条、行内码浅底;正文 16px / 行高 1.85。

## Layout Rules
**公众号编辑器会剥掉 class 和 <style>** → 一切样式必须**内联**(由 WechatRenderer 保证)。导出后一键复制富文本,粘进编辑器无需二次排版。

## Do
- 用 H2 分章、引用框立观点、列表收清单。
- 配图位用 image_slot 标出(封面/内文/图示)。

## Don't
- 不堆砌 markdown 小标题与大段加粗(交给排版节奏)。
- 不在正文里写平台不支持的复杂样式(只能内联 CSS)。

## Examples
见 studio-web `/platform-export` 公众号预览。

## Evaluation
- 首屏是否 3 秒抓人?段落是否有呼吸?重点句是否跳出?
- 复制到公众号编辑器后是否**零二次排版**?
- 读者扫 H2 是否能复述文章主线?收藏/转发理由是否明确?
