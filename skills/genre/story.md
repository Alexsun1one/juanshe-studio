# Skill: 叙事 craft(把"信息"写成"读者经历的场景")

> **来源/署名**:craft 原则蒸馏自经典写作法(场景-后续 Scene & Sequel,Dwight V. Swain《Techniques of the Selling Writer》方法论;契诃夫之枪)与 [haowjy/creative-writing-skills](https://github.com/haowjy/creative-writing-skills)(Apache-2.0)的 `prose-critique` / `llm-writing` 思路。本文件是**方法蒸馏**,非原文转载;商用前保留来源署名。

## Purpose
让叙事类内容(小说、故事化长文、案例叙事)从"AI 在叙述信息"变成"读者在经历一个场景"。本技能约束**怎么写场景**,不规定写什么剧情。

## Applies To
- 内容类型:小说章节、故事型公众号长文、知乎经验叙事、品牌故事、案例复盘里的叙事段。
- 与小说线既有「看点密集 / 对话接地 / 80-20 断章」规则**一致**,是其平台无关版本;冲突时以更具体的小说 writer 规则为准。

## Input
- 本场景目标(谁、想要什么、阻碍是什么、场景结束时发生什么改变)。
- POV 视角人物与其当前情绪。

## Output
结构化叙事正文(Content AST),遵循下面的场景结构与节奏规则。

## Rules
**场景-后续(Scene & Sequel)**
- 场景 = 目标 → 冲突 → 转折(挫败 / 代价 / 意外);不要"目标顺风顺水达成"。
- 后续 = 反应 → 两难 → 决定,把上一场景的情绪与后果消化成下一个目标。短场景可压缩,但不要全程只有场景没有后续(读者喘不过气、人物沦为工具)。

**Show, don't tell(可量化)**
- 情绪**不直接命名**:不写"他很害怕",写可观察的身体信号(指尖发凉、咽口水、退半步)。
- 每段至少 1 个**具体可感意象**(能摸 / 闻 / 看 / 听到的东西);抽象名词(希望、绝望、决心)能换成具体物件 / 动作就换。
- 用动作和对话推进,不用解释推进。

**人物声音差异化**
- 每个角色有**可辨识的说话方式**(用词、句长、口头禅、回避什么)。对白允许打断、不完整句、答非所问——真人不说工整完整句。
- 盲测:遮住"某某说",仍能从台词认出是谁。

**埋钩与兑现(契诃夫之枪)**:前面出现的具体物件 / 反常细节,后面要响;兑现时回扣读者看过的原文,不要含糊带过。

**节奏**:长短句、长短段交替;紧张处短促,沉浸处铺展;避免连续等长句 / 等长段(AI 最明显的结构痕迹)。

## Style Tokens
- 进场:从一个**具体动作或反常细节**切入,而非从背景说明切入。
- 信息**滴灌**而非倾倒:背景设定拆进动作和对话里,不开"设定讲解"段。
- 收尾:留一个具体的、未解决的张力(改变 + 新问题),而非总结陈词。

## Layout Rules
靠段落节奏分层,不靠堆小标题与大段加粗;具体平台排版交给 Renderer。

## Do
- 先定"本场景结束时世界发生了什么改变"(信息 / 关系 / 处境 / 筹码),再写。
- 让冲突有真实代价;允许 POV 人物的判断是错的。
- 每个对话密集段都用感官锚点接地。

## Don't
- 不写"全场震惊 / 所有人都…"群像统一反应——落到 1-2 个具体人的具体反应。
- 不用"他感到一阵 X"模板命名情绪;不写顺风顺水无代价的场景。
- 不开教科书式背景说明段;不让角色说工整完整的"信息播报"台词。

## Examples
弱(telling):"她非常紧张,不知道该怎么办。"
强(showing):"她把手机正面扣在桌上,又翻回来,屏幕亮着,她没看。"
(示意,非固定句)

## Evaluation
**场景自检**:有目标-冲突-转折吗?结束时有具体改变吗?
**prose 自检**:每段有具体意象吗?情绪是 showing 不是命名吗?对话能盲测认人吗?句段长短有变化吗?埋的钩子后面响了吗?
