## 题材禁忌

- 行业细节失真(IT 不写代码、广告不写策划、法律不写案件)
- 主角全能(任何项目都能 cover,任何客户都能搞定)
- 老板 / 上司 / HR 脸谱化(全部黑心 / 全部弱智)
- 用'职场厚黑学'装逼,缺乏真实可行的方法论
- 跳过日常工作只写'高光时刻',缺乏职场质感
- 同事关系简单(只有'毒舌损友 / 嫉妒同事 / 暗恋异性')
- 升职 / 涨薪用'老板赏识'解决,缺乏真实绩效

## 行业与职场规则

- 选定一个行业作为主战场(互联网 / 金融 / 咨询 / 广告 / 律师 / 医生 / 媒体 / 教育),写到行业内细节
- 公司架构真实(部门 / 职级 / 汇报线 / 项目组)
- KPI / OKR / 述职 / 季度评估 / 晋升答辩 都是真实流程
- 职场政治不是单纯'坏人 vs 好人',而是利益分配与立场博弈
- 客户 / 供应商 / 合作方 是网状关系,各有利益诉求
- 加班 / 通勤 / 健康 / 家庭 是真实压力源,不能跳过
- 不同行业有不同'潜规则'(请客送礼 / 信息差 / 关系运营)

## 工作与人物

- 每个项目要展示完整流程:需求 → 方案 → 评审 → 执行 → 交付 → 复盘
- 失败比成功更重要:第一个项目栽跟头 → 反思 → 第二个项目突破
- 老板 / 上司 / 同事 都要有立体动机:不是无理由扶持或打压
- 主角的'专业能力'要具体展示(代码 / 文案 / 谈判 / 数据 / 设计),不是'她很厉害'
- 工作之外的生活(室友 / 家人 / 朋友 / 兴趣)是必要的人情味
- 职业道德困境是高级冲突(老板让做灰色操作 / 客户要求过分需求)
- 离职 / 跳槽 / 创业 是重要节点,要有真实考量(钱、人脉、机会、风险)

## 叙事指导

第一章必须把行业、公司、岗位、主角处境交代清楚,让读者立刻入戏。
工作日常(开会 / 写文 / 改方案 / 见客户)是塑造质感的关键,不能跳过。
对话要符合职业语境:程序员有程序员的术语,律师有律师的腔调,不能一锅煮。
项目细节要让懂行的读者点头(用真实工具 / 真实流程 / 真实痛点)。
真正高水准的职场文是'让外行能看懂、内行能共鸣',而非'装逼大会'。
升职加薪不是终点;成长、独立、找到自己的人生方向 才是。
